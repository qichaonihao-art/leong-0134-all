# PROJECT_CONTEXT

## 项目名称
凉飕飕声音克隆工具

## 项目定位
这是一个给公司同事使用的在线网页工具，主要用途是：
上传一段音频样本，进行声音克隆，然后输入目标文案，用克隆出来的声音生成新的语音结果，并支持试听和下载。

这是一个偏内部使用的小工具，重点不是复杂平台化，而是：
- 页面简单直观
- 操作流程清晰
- 非技术同事也容易上手
- 能快速完成“上传样本 -> 克隆音色 -> 输入文案 -> 生成语音”这条链路

---

## 当前技术背景
这个项目最初是在 VS Code + Claude Code + claude-sonnet-4-6 的组合下逐步完成的。

后续可能会切换到新的 AI 编程组合继续开发，比如：
- VS Code
- 其他 AI 编程插件 / Agent
- GPT-5.4 等模型

所以希望新的助手在继续开发时，先理解本项目当前已经完成的内容和产品思路，再继续新增和优化功能。

## 当前已经完成的核心功能

### 1. 上传音频样本
- 用户可以上传一段音频文件作为声音样本
- 目标要求是音频时长至少 10 秒
- 上传后可以试听，方便确认上传的样本是否正确

### 2. 音色克隆
- 系统基于上传的音频样本进行声音克隆
- 当前已经打通与智谱相关接口的流程
- 文件上传时，`purpose` 的正确值是 `voice-clone-input`
- 这一点之前是排查出来的关键问题

### 3. 输入目标文案
- 用户可以输入要被朗读的文字
- 输入框已经优化为：
  - 高度可根据内容自动调整
  - 不再固定高度滚动查看

### 4. 生成语音
- 使用克隆好的声音去朗读输入文案
- 最终生成新的语音结果

### 5. 生成结果试听和下载
- 生成后的音频结果支持：
  - 在线试听
  - 下载音频文件

### 6. 同一音色连续复用
- 当前已经优化过交互逻辑
- 同一个已经克隆成功的音色，不需要每次重新上传和重新克隆
- 用户可以连续输入不同文案，持续使用当前音色生成多段语音

### 7. 音色本地保存
- 每个用户可以在自己的浏览器本地保存音色
- 不需要和其他同事共享
- 不是公司公共音色库
- 目标是每个用户在自己的浏览器里大概保存 5 个左右常用音色
- 已保存音色可以在后续继续调用

### 8. API Key 本地保存
- 网页顶部需要输入 API Key
- 已经优化为：
  - 第一次输入后保存在当前浏览器本地
  - 后续刷新页面或再次打开网页时，不需要重新输入
- 这是“每个用户在自己的浏览器中保存自己的 Key”，不是多人共享

## 当前产品逻辑

目前更合理的产品结构应该理解为 3 个区域：

### A. 声音样本与音色创建区
- 上传音频样本
- 试听原始音频
- 克隆音色
- 保存音色到本地列表

### B. 我的音色区
- 展示当前浏览器本地保存的音色
- 最多大概保留 5 个左右
- 用户只能看到自己的音色
- 不需要做多人共享
- 可以选择某个已有音色继续生成语音
- 可以删除、替换不需要的音色

### C. 文案生成区
- 选择当前要使用的音色
- 输入目标文案
- 生成语音
- 试听结果
- 下载结果
- 输入框内容在生成后不会自动清空，便于边听边对照文案

## 存储设计原则

### 当前原则
本项目优先采用“浏览器本地存储”的方案，而不是复杂的后端数据库。

原因：
- 每个同事只使用自己的音色和自己的 Key
- 不需要多人共享音色
- 项目目标是先做成一个简单、稳定、能用的内部工具
- 方便部署到 GitHub + Vercel

### 已知边界
由于当前是本地存储方案，因此：
- 换浏览器可能看不到之前保存的数据
- 换电脑也可能没有
- 清理浏览器缓存后，数据可能丢失
- 这是当前版本可以接受的取舍

## 开发流程说明
当前项目的常规更新流程是：

1. 在本地使用 VS Code / AI 工具修改代码

3. 提交到 GitHub

5. 刷新线上网页验证效果

## 后续继续开发时的要求

新的 AI 助手在继续开发本项目时，请遵循下面这些原则：

### 1. 优先保持简单
不要一上来做得太复杂，优先保证工具好用、流程清楚、同事容易上手。

### 2. 优先理解已有逻辑
不要把当前项目重新做成另一套复杂平台，要在现有产品逻辑上继续优化。

### 3. 不要默认做多人共享系统
当前项目不是公共音色平台，而是“每个用户在自己浏览器中管理自己的音色”。

### 4. 保持页面直观
页面重点是：
- 流程清晰
- 非技术人员易理解
- 每一步有明确状态提示
- 试听和下载入口明显

### 5. 修改时注意现有体验
当前已经优化过这些交互，不要随便改坏：
- 文案输入框自动增高
- 生成完成后不清空文案
- API Key 本地保存
- 已克隆音色可连续复用
- 本地音色列表可继续使用

## 如果继续新增功能
继续新增功能时，建议优先按下面顺序考虑：

1. 是否符合“内部工具、简单易用”的定位
2. 是否会破坏当前已经跑通的闭环
3. 是否真的需要后端
4. 能否优先用前端 + 本地存储完成
5. 是否方便同事理解和使用
